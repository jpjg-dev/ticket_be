# 동시성 테스트 계획서

## 목적

현재 예매 시스템에서 같은 좌석에 대해 동시에 예약 요청이 들어올 때  
다음 항목을 확인한다.

- 중복 예약이 발생하는지
- 실패율이 어느 정도인지
- 성공 요청이 몇 건인지
- 좌석 상태가 일관되게 유지되는지

---

## 테스트 범위

이번 테스트는 **좌석 예약 동시성 문제만** 확인한다.

확인 대상:
- `createReservation()`
- 동일한 `seatId`에 대한 동시 요청

제외 대상:
- 결제 승인
- 중복 결제
- 결제 실패
- 환불/취소 정책

---

## 테스트 시나리오

### 시나리오 1
- 같은 `seatId`
- 여러 명의 사용자
- 동시에 예약 요청

예시:
- 좌석 1개
- 사용자 20명
- 동시에 `createReservation()` 호출

---

## 확인 항목

### 1. 성공 건수
- 총 몇 건의 예약이 성공했는지 확인

### 2. 실패 건수
- 총 몇 건이 실패했는지 확인

### 3. 실패율
- 실패 건수 / 전체 요청 수

### 4. 중복 예약 발생 여부
- 동일한 좌석으로 Reservation이 2건 이상 생성되었는지 확인

### 5. 좌석 최종 상태
- 최종적으로 Seat 상태가 어떤 값인지 확인

---

## 기대 결과

정상이라면:
- 1건만 성공
- 나머지는 실패
- 동일 좌석에 대한 중복 예약 없음

문제가 있다면:
- 2건 이상 성공
- 동일 좌석에 대해 여러 Reservation 생성
- 중복 예약 발생

---

## 테스트 방법

JUnit 기반 멀티스레드 테스트 사용

사용 도구:
- `ExecutorService`
- `CountDownLatch`
- `AtomicInteger`
